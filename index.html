02

Native CSS Carousel Controls

Craig · What

Native CSS Carousel Controls

::scroll-button(), ::scroll-marker, and scroll-marker-group let CSS add carousel buttons and marker navigation to a scroll container. The key benefit is that the browser creates stateful controls with roles, tab order, focus behaviour, and active states, instead of leaving all of that work to custom JavaScript.

What is it?

A CSS feature set for adding browser-generated scroll buttons and marker navigation to a scroll container.

What does it do?

::scroll-button(left) and ::scroll-button(right) create previous and next buttons. scroll-marker-group and ::scroll-marker create dot or label navigation.

Why use it?

It can reduce custom JavaScript while keeping the carousel as a normal scroll area. Because the controls are browser-generated, accessibility details such as keyboard access, roles, state, and focus behaviour are handled more consistently.

Where would you use it?

Product galleries, card shelves, image galleries, onboarding panels, feature tours, and any simple manual carousel.

What

Craig · How

How the new CSS builds up

Native CSS carousel controls start with a normal scroll container. Then CSS adds browser-generated controls around that scroller: ::scroll-button(left), ::scroll-button(right), scroll-marker-group, ::scroll-marker, and :target-current.

  1. Start with a scroller. Use display: flex and overflow-x: auto so the square tiles become a horizontal scroll area.
  2. Add scroll snap. Use scroll-snap-type: x mandatory on the scroller and scroll-snap-align: center on each tile so the carousel stops cleanly.
  3. Add scroll buttons. Use ::scroll-button(left) and ::scroll-button(right) so the browser creates previous and next buttons.
  4. Add button content. Use content to place the visible arrows and accessible fallback labels, such as "⬅" / "Scroll Left".
  5. Add scroll markers. Use scroll-marker-group: after to create the marker area, then use ::scroll-marker on the tiles to create dot navigation.
  6. Style the current marker. Use :target-current so the active dot changes when that tile is current.
  7. Combine the controls. The final demo uses the scroller, snap points, side buttons, and dot markers together without custom JavaScript.
How

Craig · Demo

Build-up demo

The demo builds the carousel in stages: first a normal horizontal scroller, then scroll snap, then browser-generated previous/next buttons, then browser-generated dot markers, and finally both controls together. The outlined square tiles make the scrolling behaviour easy to see.

Step 1

Start with a scroller

The base pattern is just a horizontal list with overflow and scroll snap. This still works even if the new carousel controls are unsupported. 

.carousel {
        display: flex;
        gap: 1rem;
        overflow-x: auto;
        scroll-snap-type: x mandatory;
      }

      .carousel > li {
        flex: 0 0 10rem;
        width: 10rem;
        height: 10rem;
        border: 3px solid currentColor;
        scroll-snap-align: center;
      }

Step 2

Add scroll buttons

::scroll-button(left) and ::scroll-button(right) let CSS ask the browser to create previous and next buttons for the scroll area. These browser-created buttons page through most of the scroll area.

.carousel::scroll-button(left) {
        content: "⬅" / "Scroll Left";
      }

      .carousel::scroll-button(right) {
        content: "⮕" / "Scroll Right";
      }

      .carousel::scroll-button(*):focus-visible {
        outline-offset: 5px;
      }

Step 3

Add scroll markers

scroll-marker-group creates the marker area, and ::scroll-marker creates a marker for selected scroll items. :target-current styles the active marker. 

.carousel {
        scroll-marker-group: after;
      }

      .carousel > li::scroll-marker {
        content: " ";
      }

      .carousel > li::scroll-marker:target-current {
        background: currentColor;
      }

Step 4

Use both together

The full version combines the basic scroller, scroll snap, buttons, and markers. The new controls are wrapped in support checks so unsupported browsers still get the horizontal scroller. 

.carousel {
        scroll-snap-type: x mandatory;
        scroll-marker-group: after;
      }

      .carousel::scroll-button(left) {
        content: "⬅" / "Scroll Left";
      }

      .carousel::scroll-button(right) {
        content: "⮕" / "Scroll Right";
      }

      .carousel > li::scroll-marker {
        content: " ";
      }
